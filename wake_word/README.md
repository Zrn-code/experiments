# Wake Word Detection — 本地喚醒詞偵測

## 目標

建立一個**完全離線、本地端運行**的中文喚醒詞（Wake Word）偵測功能，讓使用者可以自訂喚醒詞（如「小蟬」、「早安」等中文詞語），當系統偵測到該詞語時自動觸發開關（例如啟動語音點餐流程）。

此功能最終需整合至 Flutter App，並且**至少支援 Android 平台**。

## 核心需求

| 項目 | 說明 |
|------|------|
| 離線運行 | **禁止**使用任何第三方 API 或遠端服務，模型推論必須完全在裝置本地執行 |
| 中文支援 | 喚醒詞為**中文詞語**（2–4 字），需處理中文語音特性（聲調、連音等） |
| 自訂喚醒詞 | 使用者可自行設定喚醒詞，系統需能針對新詞重新訓練/微調模型 |
| Flutter 整合 | 最終以 Flutter Plugin 或其他形式提供給 **Flutter App 使用** |
| Android 支援 | 至少需支援 Android 平台（API 26) |
| 低功耗 | 需持續監聽麥克風，模型需足夠輕量以在行動裝置上長時間運行 |


## 驗收標準

- [ ] 完成技術選型，確定模型架構與訓練方案
- [ ] 可偵測中文喚醒詞的模型（如「小蟬」、「早安」）
- [ ] 可以設定新的喚醒詞
- [ ] 準確率指標：喚醒率 > 90%、誤喚醒率 < 5%（安靜環境）
- [ ] 建立 Flutter Plugin或其他方式，透過 Platform Channel 呼叫 Android 端推論
- [ ] Android Demo App 可持續監聽並在偵測到喚醒詞時觸發回呼
- [ ] 使用者可透過 App 設定新的喚醒詞（含引導錄音流程）
- [ ] 提供說明文件
